# Update Docs After Task

After completing a task or before finishing a session:

1. **LEARNINGS.md** -- Add non-obvious discoveries (gotchas, patterns, design decision rationale).
2. **TODO.md** -- Add any unfinished work, follow-ups, or surfaced problems to the appropriate priority section.
3. **SHIFT_CHART.md** -- After significant work, update Current State, Open Questions, and Active Risks. Update the "Last Updated" line with today's date and your agent name. Re-rate the Confidence Level honestly.
4. **DECISIONS.md** -- If you made a product or architecture choice a future agent might question, add a date-stamped entry with 2-3 sentences of reasoning.
5. **CLAUDE.md / DESIGN.md / README.md** -- If priorities, product ideas, design paradigm, or architecture changed, update these to reflect the current state.
