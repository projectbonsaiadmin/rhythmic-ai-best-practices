# Read Docs Before Coding

Before writing any code:

1. Read `CLAUDE.md` and `DESIGN.md`.
2. Read the spec or PRD for the feature you're working on (check `docs/` directory).
3. If the task touches routing or layouts, read the main router file and the feature's layout component.
4. If it involves a specific feature, read that feature's layout and main page component.
5. Check `TODO.md` -- see if your task is already tracked.
6. Check `LEARNINGS.md` -- don't repeat solved problems.
7. Read `SHIFT_CHART.md` -- know what other agents are working on to avoid conflicts.
8. Read `DECISIONS.md` -- don't "improve" a deliberate choice without discussing it first.

For planning, include a confidence rating (1-10) for each large or complicated task and a "questions for lead engineer" section.
